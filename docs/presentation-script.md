# Peak FTP Observatory 발표 대본 (동일 네트워크 FTP · IP/동시접속 관제)

> 상황: 동일 네트워크에서 접속하는 학생들의 FTP 업로드/웹 요청을 관제. 클라이언트 IP를 자동 기록하고, RPS·지연·오류율·CPU/메모리와 동시 접속 추정치를 실시간 표시. 발표 시간 7~8분.

## 0. 오프닝 (30초)
- “지금 보시는 대시보드는 동일 네트워크에서 접속하는 기기의 IP를 자동 기록해 FTP 업로드와 웹 요청을 실시간으로 분석합니다.”
- “IIS/FileZilla/Node 어디서든 접속하면 IP가 로그에 남고, 처리량·지연·오류율, CPU/메모리, 동시 접속 수를 한눈에 볼 수 있습니다.”

## 1. 상단 안내 & 자동 IP 기록 (40초)
- 포인트: Hero 영역(현재 단계 칩, IP 칩), IP 로그 섹션.
- 멘트:
  - “접속 즉시 오른쪽 칩에 제 IP가 잡힙니다. ‘IP 접속 로그’에는 최근 IP/시간/메모/User Agent와 동시 접속 추정치가 표시됩니다.”
  - “IP는 `/api/whoami`, 접속 기록은 `/api/visit`, 로그는 `/api/iplog`로 가져오며 2~8초 간격으로 새로고침됩니다.”

## 2. 메트릭 카드/차트 소개 (50초)
- 포인트: 업로드 처리량·지연, 오류율, CPU/메모리, 동시 접속 카드 / 두 개의 실시간 차트.
- 멘트:
  - “카드에는 최근 15초 이동창으로 계산한 RPS, 평균 지연, 오류율이 표시되고, 동시 접속은 최근 2분간 고유 IP를 추정해 보여줍니다.”
  - “차트 왼쪽은 Upload Throughput vs Response, 오른쪽은 CPU·메모리입니다. 2초마다 `/api/metrics`로 실측치를 받아옵니다.”

## 3. 단계 전환 시연 (Low → Normal → Peak → Overload, 3분)
- 버튼 시연:
  - Low: “베이스라인입니다. 리소스/지연이 안정적이고 경고 없음.”
  - Normal: “CPU/메모리 조금 올리고, 실제 업로드나 k6 요청을 넣으면 그래프가 즉시 반응합니다.”
  - Peak: “CPU 워커·메모리 압박을 더 키워 지연과 오류율이 올라가고, 경고 배너가 뜹니다.”
  - Overload: “의도적 포화 상태. RPS/지연/오류율이 크게 변하고, 안정성 점수가 떨어집니다.”
- 멘트:
  - “각 단계 전환은 `/api/load/:level`에 POST, `/api/metrics` 폴링으로 반영됩니다.”
  - “Task Manager/PerfMon을 옆에 두고 CPU·메모리·디스크 IO 스파이크가 실제로 일치하는지 확인해 주세요.”

## 4. 동시 접속 & IP 로그 강조 (50초)
- 포인트: IP 로그 테이블, 동시 접속 카드.
- 멘트:
  - “여러 기기에서 동시에 접속하면 IP 로그에 순차로 쌓이고, 고유 IP 수를 기준으로 동시 접속 카드가 늘어납니다.”
  - “교육용이라 IP만 저장하며, 선택적으로 메모를 남겨 팀/좌석 정보를 태깅할 수 있습니다.”

## 5. 리더보드 (40초)
- 포인트: IP/메모 기반 리더보드.
- 멘트:
  - “리더보드는 IP/메모별 최고 RPS, 평균 지연, 오류율, 안정성 점수를 정렬해 보여줍니다.”
  - “닉네임·메모를 저장하면 동일 IP에 태그가 붙어 누가 어떤 성능을 냈는지 바로 비교할 수 있습니다.”

## 6. CyberAttack Sim 짧게 언급 (30초)
- “`/service.html`에서 DDOS/Bruteforce/PortScan/SQLi를 가상으로 날려 애니메이션과 로그를 볼 수 있습니다. 실서비스에는 영향 없이 학습용 시뮬레이션입니다.”

## 7. 정리 (30초)
- “동일 네트워크 접속 IP를 자동 기록하고, 단계별 부하/실시간 지표/동시 접속을 합쳐 FTP 업로드 환경을 투명하게 보여줍니다.”
- “실제 FTP 로그 파서나 k6/AB 트래픽을 연결하면 훨씬 현실적인 성능 실습이 가능합니다. 지금부터 함께 버튼을 눌러 보겠습니다.”
