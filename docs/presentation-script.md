# Windows Server 2016 동적 부하 데모 발표 대본 (실측 기반)

> 상황: Windows Server 2016 VM 위에서 Express 서비스 실행. 모든 지표는 Task Manager/PerfMon과 동일한 실시간 값(CPU/메모리/디스크 IO, 실제 요청 RPS/지연/오류율)으로 표시됩니다. (예상 6~8분)

## 0. 오프닝 (30초)
- “지금 보시는 화면은 Windows Server 2016 VM 위에 올린 동적 부하 데모입니다. CPU·메모리·디스크 IO가 실시간으로 계측돼 Task Manager/PerfMon과 똑같이 보입니다.”
- “왼쪽은 웹 대시보드, 오른쪽은 Task Manager/PerfMon으로 실제 리소스 변화를 함께 보겠습니다.”

## 1. 현재 상태 소개 (30초)
- 화면 포인트: 상단 Hero 영역 – “Win2016 Load Lab”, 버튼 4개(Low/Normal/Peak/Overload), 현재 단계 칩.
- 말하기: “기본값은 Low 단계입니다. CPU/메모리 카드가 안정적이고, RPS·지연은 실제 요청이 들어올 때만 증가합니다. 아래 ‘실시간 시스템 상태’에 현재 CPU/메모리/RPS/지연/오류율이 그대로 표시됩니다.”

## 2. Low → Normal 전환 (1분)
- 클릭: “Normal 부하” 버튼.
- 멘트:
  - “Normal은 평균 부하 단계입니다. CPU 워커 1개, 메모리 64MB를 실제로 점유합니다.”
  - “2초마다 `/api/metrics`를 폴링해 실측 RPS, 평균 지연(최근 15초), 오류율, CPU/메모리, 디스크 IO를 그대로 보여줍니다.”
- 보여주기: Task Manager CPU가 이전 대비 올라가는 것, 메모리 사용량 소폭 증가. (필요 시 브라우저 새로고침을 반복하거나 k6/AB로 실제 요청을 넣어 RPS/지연 변화를 강조)

## 3. Normal → Peak 전환 (1분)
- 클릭: “Peak 부하” 버튼.
- 멘트:
  - “Peak는 CPU 워커를 추가로 돌리고, 메모리 압박을 128MB까지 올립니다. 지표는 전부 실측값입니다.”
  - “temp 파일 IO 버스트가 주기적으로 발생해 디스크 그래프가 순간적으로 튑니다. Task Manager/PerfMon 디스크 Active Time을 함께 보여주세요.”
  - “경고 배너는 응답 지연 300ms+, 오류율 1.5%+, CPU 85%+에서 뜹니다. RPS·지연은 실제 트래픽에 따라 움직입니다.”
- 보여주기: 경고 배너가 뜨는지 확인, 차트 곡선이 위로 상승하는 모습, PerfMon/Task Manager CPU·Disk 그래프 스파이크.

## 4. Peak → Overload 전환 (1~2분)
- 클릭: “Overload 과부하” 버튼.
- 멘트:
  - “Overload는 의도적으로 포화 상태를 만듭니다. CPU 워커를 더 늘리고, 메모리 압박 256MB, IO 버스트 간격을 더 짧게 둡니다.”
  - “실제 서비스가 견딜 수 있는 한계를 보여주며, 경고 배너·리더보드가 실측 값에 맞춰 어떻게 변하는지 확인합니다. RPS/지연/오류율은 실제 요청이 많을수록 더 크게 변합니다.”
- 보여주기:
  - Task Manager CPU 90% 이상, 메모리 여유가 줄어드는 모습, 디스크 Active Time 증가.
  - 대시보드 카드/차트에서 응답 시간 상승, 오류율 숫자 상승 확인.

## 5. 리더보드 & 닉네임 (30초)
- 입력: 닉네임 입력 후 “저장”.
- 멘트:
  - “닉네임을 저장하면 현재 세션 성능 기록이 리더보드에 적재됩니다. Peak RPS, 평균 지연, 오류율, 안정성 점수 순으로 정렬됩니다.”
  - “동시에 여러 사람이 접속해도 15초 이동 창으로 계산하기 때문에 실시간 변화를 비교할 수 있습니다.”

## 6. 정리 (30초)
- “Low→Normal→Peak→Overload 4단계에서 CPU 워커·메모리 압박·IO 버스트만 설정해두고, 지표는 모두 실측 값으로 표시했습니다.”
- “Task Manager/PerfMon 지표와 대시보드 지표가 동일하게 움직이는지 확인하며 CPU·메모리·IO 한계를 체감했습니다.”
- “k6/AB/JMeter 등 실제 트래픽을 넣으면 RPS·지연·오류율이 즉시 반영되어 더 현실적인 테스트가 가능합니다.”

## 7. Q&A 대비 포인트 (추가 멘트용)
- “부하 레시피는 `src/server/loadManager.ts`에서 CPU 워커 수, 메모리 압박량, IO 주기를 조정할 수 있습니다. RPS/지연/오류율은 전적으로 실제 요청에 의해 계산됩니다.”
- “메모리 사용/여유는 실시간 계측이며, IO 버스트는 temp 파일 쓰기/읽기로 발생시킵니다.”
- “과부하 시나리오를 더 강하게 하려면 Overload 프로필의 busyMs/idleMs, memoryPressureMb, ioIntervalMs를 조정하면 됩니다.”
