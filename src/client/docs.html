PeakLoadVisualizer · Docs

폴더/파일 구조 & 학습 가이드

리포지토리: GitHub

빠른 시작

npm install
npm run build
npm start    # http://localhost:3000
# 개발 모드: npm run dev  (ts-node + 클라이언트 watch)

부하 도구(k6/AB/JMeter)로 요청을 보내면서 대시보드/리더보드가 갱신되는지 확인하세요.

아키텍처 한눈에 보기

  • 서버: Node.js(Express) + TypeScript, `/api/metrics`에서 실측 CPU/메모리와 요청 지연/오류율 제공.
  • 클라이언트: HTML + TS(빌드 후 JS), Chart.js로 RPS/지연/CPU/메모리 시각화, 리더보드 표시.
  • 부하: k6/AB/JMeter 중 택1로 RPS/지연/오류율을 발생시켜 학습·비교.
  • 모니터링: Performance Monitor / Task Manager를 병행해 실제 자원 사용을 관찰.

루트

  • README.md — 전체 개요, 실행 방법, API 설명.
  • package.json, tsconfig*.json — 의존성/빌드/TypeScript 설정.
  • docs/ — 시나리오/윈도우 세팅/부하 계획 문서 원본.
  • src/ — TypeScript 소스(서버·클라이언트).
  • public/ — 빌드된 정적 자산(JS/CSS).
  • dist/ — 서버 빌드 산출물.

서버 (src/server)

  • app.ts — Express 엔트리: 정적 서빙, `/api/metrics`(실시간 메트릭), `/api/load/:level`(부하 단계), `/api/leaderboard`, `/docs.html` 라우트.
  • metrics.ts — systeminformation으로 CPU/메모리 실측, 요청별 지연·오류율을 15초 윈도우로 집계, 경고 메시지 생성.

클라이언트 (src/client)

  • index.html — 대시보드 레이아웃, 부하 단계 버튼, 리더보드 테이블, 툴팁/언어 토글, Footer 링크.
  • styles.css — 테마, 카드/패널/차트/툴팁 스타일.
  • main.ts — 메트릭 폴링(2s), 부하 전환, 닉네임 로컬 저장, 리더보드 렌더링, i18n, 도움말 토글.
  • docs.html — 이 문서 페이지.

문서 (docs/)

  • overview.md — 기능/스택/실습 아이디어 요약.
  • windows-setup.md — VMware + Windows Server 2016 설치 및 배포 가이드.
  • load-plan.md — Low/Normal/Peak 단계 정의, 메트릭 관측 포인트, 로드툴 예시.

실습 체크리스트

  • 1) 서버 실행 후 /api/metrics 응답이 갱신되는지 확인.
  • 2) 페이지에서 닉네임 저장 → 리더보드에 기록되는지 확인.
  • 3) Low → Normal → Peak 버튼 클릭 시 지표 변화와 경고 배너 확인.
  • 4) k6/AB/JMeter로 부하를 주며 PerfMon/Task Manager를 병행 관찰.
  • 5) GitHub 코드 열람 후, 스스로 엔드포인트/차트/리더보드 로직을 변경해보기.

추천 학습 주제

  • RPS와 Latency의 상관관계, 오류율이 1%→5%로 증가할 때 사용자 경험이 어떻게 달라지는지.
  • Peak 상황에서 CPU/메모리 포화가 응답 지연에 미치는 영향.
  • 로드툴(k6/AB/JMeter)별 장단점과 시나리오 설계 방법.
  • IIS 역프록시 vs 순수 Node.js 호스팅의 차이.
