닉네임 설정 (IP 기준)

실습자가 접속하면 자신의 IP에 매핑된 닉네임을 설정하고, 부하 테스트 결과가 리더보드에 기록됩니다. 프라이버시를 위해 기본값은 익명 닉네임으로 시작합니다.

기록 방식

  • IP별 15초 이동창으로 RPS/지연/오류율 계산
  • CPU/메모리 실측(`systeminformation`) 반영
  • 최고 RPS, 최저 평균 지연, 최소 오류율, 안정성 점수 저장

단계별 부하 실습

버튼을 눌러 서버 부하 단계를 전환하면 `/api/load/:level`이 호출되고, `/api/metrics`가 갱신됩니다. 실제 발표에서는 k6/AB/JMeter 실행을 여기에 연결할 수 있습니다.

  • Low: 캐시 워밍업, 정상 지표 확인
  • Normal: 평균 응답 시간과 CPU 안정성 확인
  • Peak: 지연/오류율 상승 시 경고 배너 확인

모니터링 포인트

  • Performance Monitor: % Processor Time, Available MBytes
  • 로드툴: k6/AB/JMeter 결과와 그래프 비교
  • Task Manager 캡처로 실시간 리소스 공유
